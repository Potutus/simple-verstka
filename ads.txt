Строка содержит домен рекламной платформы, идентификатор партнера и тип отношений с продавцом.

Пример: example.com, 1234, DIRECT.

Типы отношений с продавцом могут быть:

DIRECT (прямые) — рекламные сети, с которыми вы сотрудничаете напрямую;
RESELLER (посреднические) — партнер использует сторонних посредников для продажи инвентаря.
Мы не рекомендуем добавлять в код информацию о компаниях, с которыми вы не работаете.


Если у вас нет файла ads.txt, создайте его. Чтобы файл ads.txt можно было проверить, он должен обязательно содержать идентификатор издателя в правильном формате.

Создайте текстовый файл (.txt) с помощью редактора обычного текста, например Notepad (Windows) или TextEdit (Mac).
Войдите в аккаунт AdSense.
Если на главной странице есть оповещение, нажмите Исправить. В ином случае, выберите Сайты, а затем нажмите на название сайта со статусом файла ads.txt "Не найден".
Примечание. Для некоторых сайтов в зависимости от статуса файла ads.txt может быть доступно несколько вариантов подтверждения. Выберите Фрагмент кода Ads.txt.
Нажмите Копировать.
Скопируйте строку в каждый файл ads.txt.
Файл ads.txt должен выглядеть следующим образом (вместо значения pub-0000000000000000 укажите свой идентификатор издателя):

google.com, pub-0000000000000000, DIRECT, f08c47fec0942fa0

(Необязательно) Если вы используете другую рекламную сеть, ее также нужно добавить в файл ads.txt. Свяжитесь с представителями этой сети, чтобы получить необходимую информацию.
Загрузите файл ads.txt в корневой каталог вашего сайта.

Корневой каталог сайта – это каталог или папка, следующая за доменом верхнего уровня. Пример: для example.com/ads.txt корневым каталогом или папкой является example.com.

Совет. Если вы не знаете, куда добавлять файл ads.txt, найдите подробные инструкции в используемом вами конструкторе или редакторе сайтов. Также можно обратиться за помощью к поставщику услуг хостинга. Подробнее о корневом каталоге…
Чтобы проверить, правильно ли опубликован файл ads.txt, попробуйте получить доступ к его содержимому. В веб-браузере перейдите по URL файла (например, https://example.com/ads.txt). Если вы увидите содержимое файла, то, скорее всего, AdSense сможет его найти.
